### 3.3 レスポンス仕様

#### 正常時

**ステータスコード：** {{ SUCCESS_STATUS }}

```json
{
  "data": {
    "{{ FIELD_1 }}": "{{ VALUE_1 }}"
  }
}
```

一覧取得時:

```json
{
  "data": [],
  "pagination": {
    "page": 1,
    "per_page": 10,
    "total": 0,
    "total_pages": 0
  }
}
```

#### 異常時（RFC7807 互換）

| ステータスコード | 説明 |
|------------------|------|
| 400 | バリデーションエラー |
| 401 | 未認証（管理 API） |
| 403 | CSRF 不一致 |
| 404 | リソース未存在 |
| 405 | メソッド不許可 |
| 406 | Accept ヘッダー不正 |
| 500 | サーバーエラー |

```json
{
  "type": "about:blank",
  "title": "Bad Request",
  "status": 400,
  "detail": "{{ ERROR_DETAIL }}",
  "instance": "/api/{{ API_TYPE }}/{{ PATH }}.php",
  "traceId": "{{ TRACE_ID }}"
}
```

### 3.4 処理概要

1. リクエスト受信
2. `Accept` ヘッダー検証
3. 認証・CSRF 確認（管理 API）
4. バリデーション実施
5. 業務処理実行
6. レスポンス生成
7. ログ出力・監査記録

### 3.5 実装上の注意

- [ ] N+1対策
- [ ] 冪等性の確認
- [ ] トランザクション管理
- [ ] 公開 API では `is_published = 1` のみ返却
- [ ] 管理 API では PII の露出範囲を確認
- [ ] 公開 GET に `Cache-Control: public, max-age=60` を付与（該当時）
- [ ] ブラウザ呼び出し時、管理 API は `credentials: 'include'`
