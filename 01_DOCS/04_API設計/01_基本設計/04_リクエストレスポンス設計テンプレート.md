# リクエストレスポンス設計テンプレート

> APIの入力・出力形式を統一するためのテンプレートです。
>
> 共通仕様: `01_DOCS/wiki/04_API設計/00_共通仕様.md`

---

## 1. 共通方針

- [ ] JSON 形式（`Content-Type: application/json; charset=UTF-8`）
- [ ] 日付形式: `YYYY-MM-DD`（date）、`YYYY-MM-DD HH:mm:ss`（datetime）
- [ ] null の扱いを記載する
- [ ] 文字コード: UTF-8
- [ ] 項目名: snake_case
- [ ] バージョニング: `Accept: application/vnd.astrohp+json;version=1`

---

## 2. 共通リクエスト仕様

### 2.1 共通ヘッダー

| ヘッダー名 | 必須 | 説明 | 例 |
|------------|------|------|----|
| Accept | **必須** | API バージョン指定 | `application/vnd.astrohp+json;version=1` |
| Content-Type | 変更系必須 | 送信データ形式 | `application/json` |
| X-CSRF-Token | 管理 API 変更系必須 | CSRF トークン | `<64文字hex>` |
| Cookie | 管理 API 必須 | セッション Cookie | `astrohp_admin=...`（自動送信） |

> Bearer トークン（`Authorization` ヘッダー）は **使用しない**（廃止済み）。

### 2.2 ブラウザからの呼び出し

| 系統 | credentials | 備考 |
|------|-------------|------|
| 公開 API | 不要 | `Accept` のみ |
| 管理 API | `'include'` 必須 | Cookie 送受信 |

### 2.3 パラメータ種別
- [ ] クエリパラメータ（`page`, `per_page`, `from`, `to`, `id` 等）
- [ ] ボディパラメータ（POST/PUT/PATCH/DELETE）
- [ ] ヘッダーパラメータ（Accept, X-CSRF-Token）

---

## 3. リクエストボディ定義

### 3.1 作成系（管理 API POST）

```json
{
  "{{ FIELD_1 }}": "{{ VALUE_1 }}",
  "{{ FIELD_2 }}": "{{ VALUE_2 }}",
  "is_published": true
}
```

### 3.2 更新系（管理 API PUT/PATCH）
- [ ] 部分更新（指定項目のみ更新）
- [ ] `id` はクエリ `?id=1` または JSON body で指定
- [ ] 更新対象フィールドが空 → `400`
- [ ] 存在しない id → `404`

### 3.3 一覧取得系

| パラメータ | 公開 API | 管理 API |
|-----------|----------|----------|
| `page` デフォルト | 1 | 1 |
| `per_page` デフォルト | 10 | 20 |
| `per_page` 上限 | 50 | 100 |
| `from` / `to` | 任意（`YYYY-MM-DD`） | 任意（`YYYY-MM-DD`） |

---

## 4. バリデーション

| 項目 | 型 | 必須 | 制約 | 例 |
|------|----|------|------|----|
| {{ FIELD_1 }} | {{ TYPE_1 }} | {{ REQUIRED_1 }} | {{ RULE_1 }} | {{ EXAMPLE_1 }} |
| title | string | 作成時必須 | 空文字不可、255 文字以内 | `新機能追加` |
| changed_at | string(date) | 作成時必須 | `YYYY-MM-DD` のみ | `2026-06-01` |
| is_published | boolean | 任意 | `true`/`false` | `true` |
| page | integer | 任意 | 1 以上 | `1` |
| per_page | integer | 任意 | 系統ごとの上限内 | `10` |

---

## 5. 共通レスポンス仕様

### 5.1 単体成功レスポンス例

```json
{
  "data": {
    "id": 1,
    "{{ FIELD_1 }}": "{{ VALUE_1 }}"
  }
}
```

### 5.2 一覧レスポンス例

```json
{
  "data": [
    {
      "id": 1,
      "{{ FIELD_1 }}": "{{ VALUE_1 }}"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 10,
    "total": 42,
    "total_pages": 5
  }
}
```

### 5.3 作成成功レスポンス例（201）

```json
{
  "data": {
    "id": 5
  }
}
```

### 5.4 更新・削除成功レスポンス例（200）

```json
{
  "data": {
    "id": 1,
    "updated": true
  }
}
```

```json
{
  "data": {
    "id": 1,
    "deleted": true
  }
}
```

### 5.5 エラーレスポンス例（RFC7807 互換）

```json
{
  "type": "about:blank",
  "title": "Bad Request",
  "status": 400,
  "detail": "per_page must be between 1 and 50.",
  "instance": "/api/public/changelogs.php?page=1&per_page=100",
  "traceId": "9a0b2d6f8c1e4a67"
}
```

---

## 6. ステータスコード方針

| 処理 | ステータスコード | 備考 |
|------|------------------|------|
| 取得成功 | 200 | `Cache-Control` は公開 GET のみ付与可 |
| 作成成功 | 201 | `{ "data": { "id": ... } }` |
| 更新成功 | 200 | `{ "data": { "id": ..., "updated": true } }` |
| 削除成功 | 200 | `{ "data": { "id": ..., "deleted": true } }` |
| 不正リクエスト | 400 | バリデーションエラー |
| 未認証 | 401 | 管理 API |
| CSRF 不一致 | 403 | 管理 API 変更系 |
| 未検出 | 404 | |
| メソッド不許可 | 405 | |
| Accept 不正 | 406 | 全 API |
| 内部エラー | 500 | 詳細非公開 |

---

## 7. レスポンス項目設計

- [ ] `data` のスキーマを定義する
- [ ] `pagination` の共通項目（`page`, `per_page`, `total`, `total_pages`）を定義する
- [ ] 公開 API と管理 API で返却項目を分ける（例: `is_published` は管理のみ）
- [ ] NULL 許容項目を明確化する
- [ ] `is_published` は管理 API では 0/1 整数、POST 時は boolean で受け付け

---

## 8. レビュー観点

- [ ] 入出力項目が過不足ない
- [ ] Accept ヘッダー必須が明記されている
- [ ] public / admin でレスポンス項目が適切に分離されている
- [ ] 例示が RFC7807 形式と一致している
- [ ] ページネーション項目名が統一されている
