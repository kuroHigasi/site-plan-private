| ヘッダー名 | 必須 | 説明 | 例 |

| Content-Type | はい | 送信データ形式 | application/json |

## 5. 共通レスポンス仕様

### 5.1 成功レスポンス例

```json
{
  "success": true,
  "data": {},
  "meta": {
    "timestamp": "{{ TIMESTAMP }}",
    "request_id": "{{ REQUEST_ID }}"
  }
}
```

### 5.2 一覧レスポンス例

```json
{
  "success": true,
  "data": [],
  "pagination": {
    "current_page": 1,
    "per_page": 20,
    "total": 0,
    "last_page": 0
  },
  "meta": {
    "timestamp": "{{ TIMESTAMP }}",
    "request_id": "{{ REQUEST_ID }}"
  }
}
```

### 5.3 エラーレスポンス例

```json
{
  "success": false,
  "error": {
    "code": "{{ ERROR_CODE }}",
    "message": "{{ MESSAGE }}",
    "details": []
  },
  "meta": {
    "timestamp": "{{ TIMESTAMP }}",
    "request_id": "{{ REQUEST_ID }}"
  }
}
```

## 6. ステータスコード方針

| 処理 | ステータスコード | 備考 |
|------|------------------|------|
| 取得成功 | 200 |  |
| 作成成功 | 201 | Locationヘッダー要否を記載 |
| 更新成功 | 200 / 204 | 仕様に合わせて統一 |
| 削除成功 | 204 | ボディなしの場合 |
| 不正リクエスト | 400 |  |
| 認証失敗 | 401 |  |
| 権限不足 | 403 |  |
| 未検出 | 404 |  |
| バリデーションエラー | 422 |  |
| 競合 | 409 |  |
| 内部エラー | 500 |  |
