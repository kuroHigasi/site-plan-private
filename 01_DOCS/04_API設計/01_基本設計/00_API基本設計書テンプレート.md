## 4. API全体方針

### 4.1 基本方針
- [ ] エンドポイント形式（`.php` ファイル単位）を記載する
- [ ] URL 構成（`/api/public/` / `/api/admin/`）を記載する
- [ ] 命名規則を記載する
- [ ] バージョニング方針（`Accept: application/vnd.astrohp+json;version=1`）を記載する

### 4.2 共通仕様
- [ ] 必須ヘッダー（`Accept`）を記載する

- [ ] 日時形式（`YYYY-MM-DD` / `YYYY-MM-DD HH:mm:ss`）を記載する

## 5. 認証・認可設計概要

- [ ] 公開 API（認証不要・GET のみ）と管理 API（Cookie 認証）の区分を記載する
- [ ] セッション Cookie（`astrohp_admin`）の扱いを記載する
- [ ] CSRF トークン（`X-CSRF-Token`）の扱いを記載する
- [ ] Bearer トークンは **使用しない**（廃止済み）ことを明記する
- [ ] PII（email 等）は管理 API のみで返却する方針を記載する
- [ ] 管理者機能・公開サイト機能の区分を記載する

管理 API の認証・認可設計の詳細は site-plan-security リポジトリ `01_DOCS/04_API設計/01_基本設計/02_認証認可設計テンプレート.md` を参照。
