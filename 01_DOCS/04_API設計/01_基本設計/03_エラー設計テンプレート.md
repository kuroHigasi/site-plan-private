---

## 2. エラーレスポンス共通形式

```json
{
  "success": false,
  "error": {
    "code": "{{ ERROR_CODE }}",
    "message": "{{ MESSAGE }}",
    "details": []
  },
  "meta": {
    "timestamp": "{{ TIMESTAMP }}",
    "request_id": "{{ REQUEST_ID }}"
  }
}
```

### 記載ルール
- [ ] `code` は機械判定可能な値にする
- [ ] `message` は利用者向けメッセージにする
- [ ] `details` はフィールド単位のエラーに使用する
- [ ] `request_id` は追跡用に利用する

---

## 3. エラー分類

| 分類 | 代表ステータス | 説明 |
|------|----------------|------|
| バリデーションエラー | 422 | 入力値不正 |
| 認証エラー | 401 | ログイン状態不正、トークン不正 |
| 認可エラー | 403 | 権限不足 |
| 未検出エラー | 404 | リソース未存在 |
| 重複エラー | 409 | ユニーク制約違反等 |
| 制限超過エラー | 429 | レート制限超過 |
| システムエラー | 500 | 予期しない内部エラー |
