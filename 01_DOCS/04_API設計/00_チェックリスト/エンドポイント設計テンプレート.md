# API エンドポイント設計テンプレート

このテンプレートを使用してAPI エンドポイントを設計・ドキュメント化してください。

---

## テンプレート：各エンドポイント

### エンドポイント1: ユーザー一覧取得

| 項目 | 内容 |
|------|------|
| **メソッド** | GET |
| **エンドポイント** | `/api/v1/users` |
| **認証要件** | ✅ 必須（Bearer Token） |
| **アクセス権限** | `user`, `admin` |
| **説明** | ページネーション対応のユーザー一覧を取得 |

#### リクエスト

**クエリパラメータ：**
| パラメータ | 型 | 必須 | 例 | 説明 |
|-----------|------|------|-----|------|
| `page` | integer | ✗ | `1` | ページ番号（デフォルト: 1） |
| `per_page` | integer | ✗ | `20` | 1ページあたりの件数（デフォルト: 20） |
| `sort` | string | ✗ | `-created_at` | ソート（`-`は降順） |
| `filter[status]` | string | ✗ | `active` | ステータスフィルタ |
| `search` | string | ✗ | `john` | 名前またはメールアドレスでの検索 |
| `include` | string | ✗ | `roles,posts` | リレーション含有 |

**リクエスト例：**
```
GET /api/v1/users?page=1&per_page=20&sort=-created_at&filter[status]=active&include=roles HTTP/1.1
HOST: api.example.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### エンドポイント2: ユーザー新規作成

| 項目 | 内容 |
|------|------|
| **メソッド** | POST |
| **エンドポイント** | `/api/v1/users` |
| **認証要件** | ✅ 必須 |
| **アクセス権限** | `admin` |
| **説明** | 新規ユーザーを作成 |
| **ステータスコード** | 201 Created |

#### リクエスト

**リクエストボディ（JSON）：**
```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "securePassword123",
  "password_confirmation": "securePassword123",
  "status": "active",
  "role_ids": [1, 2]
}
```

**バリデーションルール：**
| フィールド | ルール | エラーメッセージ |
|----------|--------|------------------|
| `name` | 必須、文字列、最大255字 | 「名前は必須です」 |
| `email` | 必須、メール形式、ユニーク | 「有効なメールアドレスを入力してください」、「このメールアドレスは既に使用されています」 |
| `password` | 必須、最小6文字、確認フィールド一致 | 「パスワードは6文字以上である必要があります」、「パスワードが一致しません」 |
| `status` | 必須、`active`/`inactive`/`pending` | 「ステータスが不正です」 |
| `role_ids` | 配列、各値は存在するロールID | 「指定されたロールが存在しません」 |

#### レスポンス

**成功時（201）：**
```json
{
  "success": true,
  "data": {
    "id": 101,
    "name": "John Doe",
    "email": "john@example.com",
    "status": "active",
    "created_at": "2026-06-06T10:30:00Z",
    "roles": [
      {"id": 1, "name": "user"},
      {"id": 2, "name": "moderator"}
    ]
  }
}
```

**バリデーションエラー（422）：**
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "バリデーションエラーが発生しました",
    "details": [
      {
        "field": "email",
        "message": "このメールアドレスは既に使用されています"
      },
      {
        "field": "password",
        "message": "パスワードは6文字以上である必要があります"
      }
    ]
  }
}
```

**権限エラー（403）：**
```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "ユーザーを作成する権限がありません"
  }
}
```

#### 実装上の注意
- [ ] パスワードのハッシュ化（bcrypt推奨）
- [ ] メールアドレス重複チェック
- [ ] トランザクション処理（ユーザー＆ロール関連付け）
- [ ] メール確認プロセスの実装
- [ ] ロール関連付けの検証
- [ ] 監査ログの記録

### エンドポイント3: 指定ユーザー取得

| 項目 | 内容 |
|------|------|
| **メソッド** | GET |
| **エンドポイント** | `/api/v1/users/{id}` |
| **認証要件** | ✅ 必須 |
| **アクセス権限** | `user`, `admin` （自分or管理者のみ） |
| **説明** | 指定ユーザーの詳細情報を取得 |

#### リクエスト

**パスパラメータ：**
| パラメータ | 型 | 説明 |
|----------|------|------|
| `id` | integer | ユーザーID |

**クエリパラメータ：**
| パラメータ | 型 | 例 |
|----------|------|-----|
| `include` | string | `roles,posts` |

**リクエスト例：**
```
GET /api/v1/users/1?include=roles,posts HTTP/1.1
```

#### レスポンス

**成功時（200）：**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com",
    "status": "active",
    "created_at": "2026-06-01T10:00:00Z",
    "updated_at": "2026-06-06T10:30:00Z",
    "roles": [
      {"id": 1, "name": "user"}
    ],
    "posts": [
      {"id": 10, "title": "Post 1"},
      {"id": 11, "title": "Post 2"}
    ]
  }
}
```

**ユーザーが見つからない（404）：**
```json
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "ユーザーが見つかりません"
  }
}
```

### エンドポイント4: ユーザー部分更新

| 項目 | 内容 |
|------|------|
| **メソッド** | PATCH |
| **エンドポイント** | `/api/v1/users/{id}` |
| **認証要件** | ✅ 必須 |
| **アクセス権限** | 本人 or `admin` |
| **説明** | ユーザー情報の部分更新 |
| **ステータスコード** | 200 OK |

#### リクエスト

**リクエストボディ（JSON）：**
```json
{
  "name": "John Updated",
  "status": "inactive"
}
```

（必須フィールドなし、指定されたフィールドのみ更新）

#### レスポンス

**成功時（200）：**
```json
{
  "success": true,
  "data": {
    "id": 1,
    "name": "John Updated",
    "email": "john@example.com",
    "status": "inactive",
    "created_at": "2026-06-01T10:00:00Z",
    "updated_at": "2026-06-06T11:00:00Z"
  }
}
```

### エンドポイント5: ユーザー削除

| 項目 | 内容 |
|------|------|
| **メソッド** | DELETE |
| **エンドポイント** | `/api/v1/users/{id}` |
| **認証要件** | ✅ 必須 |
| **アクセス権限** | `admin` のみ |
| **説明** | ユーザーを削除（論理削除） |
| **ステータスコード** | 204 No Content |

#### リクエスト
```
DELETE /api/v1/users/1 HTTP/1.1
Authorization: Bearer token...
```

#### レスポンス

**成功時（204）：**
```
(レスポンスボディなし)
```

**権限エラー（403）：**
```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "ユーザーを削除する権限がありません"
  }
}
```
