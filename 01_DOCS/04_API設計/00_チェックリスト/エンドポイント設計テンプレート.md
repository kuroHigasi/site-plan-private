# API エンドポイント設計テンプレート

このテンプレートを使用して API エンドポイントを設計・ドキュメント化してください。

> 共通仕様: `01_DOCS/wiki/04_API設計/00_共通仕様.md`  
> 実装ガイド: `01_DOCS/04_API設計/02_詳細設計/ブラウザ開発者向けAPI利用ガイド.md`

---

## 設計の前提

| 項目 | 方針 |
|------|------|
| エンドポイント形式 | `.php` ファイル単位 |
| URL 構成 | `/api/public/`（公開） / `/api/admin/`（管理） |
| バージョニング | `Accept: application/vnd.astrohp+json;version=1` |
| 公開 API 認証 | 不要 |
| 管理 API 認証 | HttpOnly Cookie + CSRF（変更系） |
| エラー形式 | RFC7807 互換 |

## テンプレート：各エンドポイント

### エンドポイント1: 変更履歴一覧取得（公開）

| 項目 | 内容 |
|------|------|
| **系統** | public |
| **メソッド** | GET |
| **エンドポイント** | `/api/public/changelogs.php` |
| **認証要件** | 不要 |
| **CSRF** | 不要 |
| **説明** | 公開済み変更履歴をページネーション付きで取得 |

#### リクエスト

**ヘッダー：**
| ヘッダー | 必須 | 値 |
|---------|------|-----|
| Accept | ✅ | `application/vnd.astrohp+json;version=1` |

**クエリパラメータ：**
| パラメータ | 型 | 必須 | デフォルト | 制約 | 説明 |
|-----------|------|------|------------|------|------|
| `page` | integer | ✗ | `1` | 1 以上 | ページ番号 |
| `per_page` | integer | ✗ | `10` | 1〜50 | 1 ページあたり件数 |
| `from` | date | ✗ | — | `YYYY-MM-DD` | 開始日 |
| `to` | date | ✗ | — | `YYYY-MM-DD`、`from` より後は不可 | 終了日 |

**リクエスト例：**
```
GET /api/public/changelogs.php?page=1&per_page=5 HTTP/1.1
Host: localhost:8080
Accept: application/vnd.astrohp+json;version=1
```

#### レスポンス

**成功時（200）：**
```json
{
  "data": [
    {
      "id": 4,
      "title": "パフォーマンス改善",
      "body": "画面遷移のパフォーマンスを改善しました。",
      "changed_at": "2025-07-10"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 5,
    "total": 4,
    "total_pages": 1
  }
}
```

**バリデーションエラー（400）：**
```json
{
  "type": "about:blank",
  "title": "Bad Request",
  "status": 400,
  "detail": "per_page must be between 1 and 50.",
  "instance": "/api/public/changelogs.php?page=1&per_page=100",
  "traceId": "9a0b2d6f8c1e4a67"
}
```

**Accept 不正（406）：**
```json
{
  "type": "about:blank",
  "title": "Not Acceptable",
  "status": 406,
  "detail": "Unsupported API version.",
  "instance": "/api/public/changelogs.php",
  "traceId": "..."
}
```

#### 実装上の注意
- [ ] `is_published = 1` のみ返却
- [ ] `Cache-Control: public, max-age=60` を付与
- [ ] `credentials: 'include'` は不要
- [ ] N+1 クエリ対策

### エンドポイント2: 変更履歴作成（管理）

| 項目 | 内容 |
|------|------|
| **系統** | admin |
| **メソッド** | POST |
| **エンドポイント** | `/api/admin/changelogs.php` |
| **認証要件** | ✅ Cookie 必須 |
| **CSRF** | ✅ 必須 |
| **説明** | 変更履歴を新規作成 |
| **ステータスコード** | 201 Created |

#### リクエスト

**ヘッダー：**
| ヘッダー | 必須 | 値 |
|---------|------|-----|
| Accept | ✅ | `application/vnd.astrohp+json;version=1` |
| Content-Type | ✅ | `application/json` |
| X-CSRF-Token | ✅ | `<token>` |
| Cookie | ✅ | `astrohp_admin=...` |

**リクエストボディ（JSON）：**
```json
{
  "title": "新機能追加",
  "body": "説明文",
  "changed_at": "2025-09-01",
  "is_published": true
}
```

**バリデーションルール：**
| フィールド | ルール | detail 例 |
|----------|--------|-----------|
| `title` | 必須、255 文字以内 | `title is required.` |
| `body` | 任意 | — |
| `changed_at` | 必須、`YYYY-MM-DD` | `changed_at must be a valid date.` |
| `is_published` | boolean | — |

#### レスポンス

**成功時（201）：**
```json
{
  "data": {
    "id": 5
  }
}
```

**未認証（401）：**
```json
{
  "type": "about:blank",
  "title": "Unauthorized",
  "status": 401,
  "detail": "Authentication required.",
  "instance": "/api/admin/changelogs.php",
  "traceId": "..."
}
```

**CSRF 不一致（403）：**
```json
{
  "type": "about:blank",
  "title": "Forbidden",
  "status": 403,
  "detail": "CSRF token mismatch.",
  "instance": "/api/admin/changelogs.php",
  "traceId": "..."
}
```

#### 実装上の注意
- [ ] トランザクション処理
- [ ] CSRF トークン検証
- [ ] 監査ログの記録

### エンドポイント3: セッション確認（管理）

| 項目 | 内容 |
|------|------|
| **系統** | admin |
| **メソッド** | GET |
| **エンドポイント** | `/api/admin/session.php` |
| **認証要件** | Cookie（未ログインでも 200） |
| **CSRF** | 不要 |
| **説明** | ログイン状態と CSRF トークンを取得 |

#### レスポンス

**ログイン済み（200）：**
```json
{
  "data": {
    "authenticated": true,
    "admin_id": 1,
    "csrfToken": "<64文字hex>"
  }
}
```

**未ログイン（200）：**
```json
{
  "data": {
    "authenticated": false,
    "admin_id": null,
    "csrfToken": null
  }
}
```

> 未ログイン時も **401 にはならない**。

### エンドポイント4: ユーザー一覧取得（管理・PII）

| 項目 | 内容 |
|------|------|
| **系統** | admin |
| **メソッド** | GET |
| **エンドポイント** | `/api/admin/users.php` |
| **認証要件** | ✅ Cookie 必須 |
| **CSRF** | 不要 |
| **説明** | ユーザー一覧（PII 含む）を取得 |

#### レスポンス

**成功時（200）：**
```json
{
  "data": [
    {
      "id": 1,
      "name": "山田太郎",
      "email": "yamada@example.com"
    }
  ]
}
```

- ページングなし（最大 100 件、`id ASC`）
- 公開 API からは **取得不可**

## エンドポイント定義フォーマット（汎用テンプレート）

各エンドポイントについて以下を定義してください：

### 基本情報
- [ ] API 系統（public / admin）
- [ ] HTTP メソッド
- [ ] エンドポイントパス（`.php`）
- [ ] API バージョン（Accept ヘッダー v1）
- [ ] 説明

### 認証・認可
- [ ] 認証要件（不要 / Cookie）
- [ ] CSRF 要件（変更系のみ）
- [ ] 返却データの露出範囲

### リクエスト仕様
- [ ] 必須ヘッダー（Accept）
- [ ] クエリパラメータ定義
- [ ] リクエストボディスキーマ
- [ ] バリデーションルール
- [ ] サンプルリクエスト

### レスポンス仕様
- [ ] 成功時のステータスコード
- [ ] 成功時のレスポンススキーマ（`data` ラッパー）
- [ ] エラーケース（400, 401, 403, 404, 405, 406, 500）
- [ ] サンプルレスポンス（RFC7807 形式）

### ビジネスロジック
- [ ] 処理フロー
- [ ] トランザクション境界
- [ ] キャッシュ戦略（公開 GET のみ）
- [ ] 側面効果（ログ記録など）

### 実装上の注意
- [ ] ブラウザ呼び出し時の `credentials: 'include'`（管理 API）
- [ ] 公開 API の `is_published` フィルタ
- [ ] セキュリティ考慮事項
- [ ] 既知の制限事項
