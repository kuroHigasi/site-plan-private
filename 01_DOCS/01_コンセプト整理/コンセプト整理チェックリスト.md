# コンセプト整理チェックリスト

ポートフォリオサイトのコンセプトを固めるにあたり、整理・検討すべき課題をまとめたチェックリストです。

---

## 1. ターゲット・目的の明確化

- [ ] サイトの主な閲覧者は誰か（採用担当者、エンジニア仲間、一般ユーザー など）
- [ ] サイトを通じて伝えたい自分の強み・専門性は何か
- [ ] サイトの主目的は何か（就職・転職活動、スキルアピール、コンテンツ公開 など）
- [ ] サイトの副目的はあるか（ブログ、技術発信、コミュニティ形成 など）

---

## 2. コンテンツ方針

- [ ] 公開する成果物のカテゴリを確定する
    - [ ] 作成したゲームの掲載範囲（全作品 / 厳選作品）
    - [ ] 作成したUIの掲載範囲（全作品 / 厳選作品）
    - [ ] 資格学習コンテンツ（練習問題など）の掲載範囲と形式
    - [ ] 取得した資格の掲載範囲と表示方法
- [ ] 各コンテンツに必要な情報項目を整理する（タイトル、説明、使用技術、制作期間、リンク など）
- [ ] 更新頻度・更新方針を決める（随時更新 / 定期更新）
- [ ] 外部リンク（GitHub、SNS など）の掲載可否を決める

---

## 3. デザイン・ブランディング方針

- [ ] サイト全体のトーン＆マナーを決める（プロフェッショナル、カジュアル、クリエイティブ など）
- [ ] カラーパレットの方向性を決める
- [ ] フォントの方向性を決める（日本語・英語それぞれ）
- [ ] ロゴやファビコンを用意するか決める
- [ ] レスポンシブ対応の優先デバイスを決める（PC優先 / モバイル優先）

---

## 4. 技術スタック・構成方針

- [ ] フロントエンドフレームワークを選定する（React、Vue、Astro、Next.js など）
- [ ] スタイリング方法を選定する（CSS Modules、Tailwind CSS、styled-components など）
- [ ] 静的サイト生成（SSG）かサーバーサイドレンダリング（SSR）か、または静的HTMLかを決める
- [ ] ホスティング先を決める（GitHub Pages、Vercel、Netlify など）
- [ ] 独自ドメインの取得・使用可否を決める
- [ ] CMSを使うか（Markdown管理 / ヘッドレスCMS など）を決める

---

## 5. SEO・アクセシビリティ方針

- [ ] サイトの言語構成を決める（日本語のみ / 多言語対応）
- [ ] OGP（SNSシェア時のプレビュー画像・テキスト）を設定するか決める
- [ ] アクセシビリティ対応方針を決める（WAI-ARIA、キーボード操作対応 など）
- [ ] Googleアナリティクスなどのアクセス解析を導入するか決める

---

## 6. 運用・保守方針

- [ ] コンテンツの追加・更新フローを決める
- [ ] バージョン管理・デプロイフローを決める（GitHub Actions など）
- [ ] 問い合わせ窓口（メールフォーム、SNSリンク など）を設けるか決める
- [ ] プライバシーポリシーや免責事項が必要か確認する
