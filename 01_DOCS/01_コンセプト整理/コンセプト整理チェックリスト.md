# コンセプト整理チェックリスト

ポートフォリオサイトのコンセプトを固めるにあたり、整理・検討すべき課題をまとめたチェックリストです。

---

## 1. ターゲット・目的の明確化

- [x] サイトの主な閲覧者は誰か（採用担当者、エンジニア仲間、一般ユーザー など）
- [x] サイトを通じて伝えたい自分の強み・専門性は何か
- [x] サイトの主目的は何か（就職・転職活動、スキルアピール、コンテンツ公開 など）
- [x] サイトの副目的はあるか（ブログ、技術発信、コミュニティ形成 など）

---

## 2. コンテンツ方針

- [x] 公開する成果物のカテゴリを確定する
    - [x] 作成したゲームの掲載範囲（全作品 / 厳選作品）
    - [x] 作成したUIの掲載範囲（全作品 / 厳選作品）
    - [x] 資格学習コンテンツ（練習問題など）の掲載範囲と形式
    - [x] 取得した資格の掲載範囲と表示方法
- [x] 各コンテンツに必要な情報項目を整理する（タイトル、説明、使用技術、制作期間、リンク など）
- [x] 更新頻度・更新方針を決める（随時更新 / 定期更新）
- [x] 外部リンク（GitHub、SNS など）の掲載可否を決める

---

## 3. デザイン・ブランディング方針

- [x] サイト全体のトーン＆マナーを決める（プロフェッショナル、カジュアル、クリエイティブ など）
- [x] カラーパレットの方向性を決める
- [x] フォントの方向性を決める（日本語・英語それぞれ）
- [x] ロゴやファビコンを用意するか決める
- [x] レスポンシブ対応の優先デバイスを決める（PC優先 / モバイル優先）

---

## 4. 技術スタック・構成方針

- [x] フロントエンドフレームワークを選定する（React、Vue、Astro、Next.js など）
- [x] スタイリング方法を選定する（CSS Modules、Tailwind CSS、styled-components など）
- [x] 静的サイト生成（SSG）かサーバーサイドレンダリング（SSR）か、または静的HTMLかを決める
- [x] ホスティング先を決める（GitHub Pages、Vercel、Netlify など）
- [x] 独自ドメインの取得・使用可否を決める
- [x] CMSを使うか（Markdown管理 / ヘッドレスCMS など）を決める

---

## 5. SEO・アクセシビリティ方針

- [x] サイトの言語構成を決める（日本語のみ / 多言語対応）
- [x] OGP（SNSシェア時のプレビュー画像・テキスト）を設定するか決める
- [x] アクセシビリティ対応方針を決める（WAI-ARIA、キーボード操作対応 など）
- [x] Googleアナリティクスなどのアクセス解析を導入するか決める

---

## 6. 運用・保守方針

- [x] コンテンツの追加・更新フローを決める
- [x] バージョン管理・デプロイフローを決める（GitHub Actions など）
- [x] 問い合わせ窓口（メールフォーム、SNSリンク など）を設けるか決める
- [x] プライバシーポリシーや免責事項が必要か確認する
