### API共通仕様（公開 API）

本リポジトリ（site-plan-private）で管理する **公開 API（`/api/public/`）** の共通仕様である。公開 API は **ポートフォリオサイトのフロント（Astro 等）から認証なしで読み取る** ための API である。

> 管理画面から呼び出す **管理 API（`/api/admin/`）** の共通仕様・認証・CRUD は、非公開リポジトリ **site-plan-security** で管理する。管理 API の共通仕様は site-plan-security の `01_DOCS/wiki/04_API設計/00_共通仕様.md` を参照すること。

---

#### 1) ベース URL とエンドポイント構成

| 環境 | ベース URL（例） |
|------|------------------|
| ローカル Docker | `http://localhost:8080/api/` |
| 本番 | `https://<your-domain>/api/` |

エンドポイントは **`.php` ファイル単位** である（REST 風ルーティングではない）。

```
/api/public/   … 認証不要・読み取り専用（公開サイト向け。本リポジトリで管理）
/api/admin/    … 要ログイン・書き込み・PII（管理画面向け。site-plan-security で管理）
```

**旧 URL は使用しない。** 公開サイトからは以下の公開エンドポイントに差し替える。

| 旧 | 新（公開） |
|----|-----|
| `/api/changelogs.php` | `/api/public/changelogs.php` |

---

#### 2) 全リクエスト共通：必須ヘッダー

##### Accept（必須・省略不可）

```
Accept: application/vnd.astrohp+json;version=1
```

- 形式: `application/vnd.<service_name>+json;version=<major>`
- `<service_name>` はサーバー側 `API_SERVICE_NAME` と一致（省略時: `astrohp`）
- 未指定・形式不正・未対応バージョン → **406 Not Acceptable**
- 現在サポートされているバージョンは **v1 のみ**

##### Content-Type（JSON ボディを送るとき）

```
Content-Type: application/json
```

##### レスポンス形式

- 成功時: `Content-Type: application/json; charset=UTF-8`
- 一覧系: `{ "data": [...], "pagination": {...} }`
- 単体系: `{ "data": {...} }`

---

#### 3) CORS

公開 API は認証不要で、Cookie を使わないため `credentials: 'include'` は不要である。

CORS ヘッダー:

- `Access-Control-Allow-Origin: <CORS_ALLOW_ORIGIN>`
- `Access-Control-Allow-Headers: Authorization, Content-Type, Accept, X-CSRF-Token`
- `Access-Control-Allow-Methods: GET, POST, PUT, PATCH, DELETE, OPTIONS`
- `OPTIONS` リクエストは **204 No Content**（プリフライト用）

---

#### 4) 公開 API（`/api/public/`）の方針

- **認証不要**
- **GET のみ**（現状）
- **公開済みデータのみ**（例: `is_published = 1` の changelogs）
- Bearer トークンは **使わない**（廃止済み）
- email 等の PII は返さない（PII・CRUD 操作は管理 API のみ。site-plan-security 参照）

---

#### 5) エラーレスポンス（RFC7807 互換）

すべてのエラーは以下の JSON 形式で返却する。

```json
{
  "type": "about:blank",
  "title": "Bad Request",
  "status": 400,
  "detail": "page must be an integer of 1 or greater.",
  "instance": "/api/public/changelogs.php?page=abc",
  "traceId": "trace-..."
}
```

| HTTP | 意味 | 主な発生場面 |
|------|------|-------------|
| 400 | バリデーションエラー | 不正なクエリ・JSON・必須項目不足 |
| 404 | リソースなし | 存在しない・非公開の id |
| 405 | メソッド不許可 | 許可外 HTTP メソッド |
| 406 | Accept 不正 | Accept ヘッダー未設定・形式不正 |
| 500 | サーバーエラー | DB 障害等（詳細は返さない） |

`PDOException` など内部エラー詳細はクライアントに出さず、汎用メッセージで `500` を返却する。

---

#### 6) 主な環境変数

| 変数 | 説明 |
|------|------|
| `API_SERVICE_NAME` | Accept ヘッダーのサービス名（省略時: `astrohp`） |
| `CORS_ALLOW_ORIGIN` | CORS 許可オリジン（省略時: `*`） |
| `PUBLIC_API_BASE` | フロント向け API ベース URL（例: `http://localhost:8080/api`） |
| `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASS`, `DB_CHARSET` | DB 接続 |

開発時の推奨設定:

```env
PUBLIC_API_BASE=http://localhost:8080/api
```

本番:

```env
PUBLIC_API_BASE=https://your-domain.com/api
```

---

#### 7) 実装上の注意

##### やってはいけないこと

- **Bearer トークンをブラウザに埋め込む**（廃止済み）
- 公開 API 経由で **email 等の PII を期待する**（管理 API のみ。site-plan-security 参照）

##### バリデーション共通

- 日付は **`YYYY-MM-DD` のみ**（時刻不可）
- JSON body が空・不正 → `400`

##### キャッシュ

- 公開 changelogs のみ `Cache-Control: public, max-age=60`
