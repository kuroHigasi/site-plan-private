# ページ共通仕様

このドキュメントは、全ページで共通して使用されるヘッダー、パンくずリスト、フッターの仕様を定義します。

## 1. ヘッダー仕様

### 1.1 ヘッダーの役割

サイト全体のナビゲーション機能を提供し、ユーザーが任意のページへアクセスしやすくする。

### 1.2 構成要素

| 要素           | 説明             | 配置  | 備考                        |
|--------------|----------------|-----|---------------------------|
| グローバルナビゲーション | 主要ページへのリンク     | 中央  | リンク先: プロフィール、資格、作品、お問い合わせ |
| アクティブ状態表示    | 現在ページのナビリンクを強調 | ナビ上 | 下線、背景色、太字など               |

### 1.3 グローバルナビゲーション項目

| リンク名           | リンク先             | 対象ページID |
|----------------|------------------|---------|
| About          | /about/          | P002    |
| Certifications | /certifications/ | P005    |
| Works          | /works/          | P006    |
| Contact        | /contact/        | P003    |

#### 1.3.1 補足
- アクティブ状態（下位ページ訪問時）の扱い:
  - P005-01, P005-02, P005-03 訪問時は「Certifications」を強調
  - P006-01, P006-02, P006-03 訪問時は「Works」を強調

### 1.4 デザイン・レスポンシブ

- **PC**: 通常ヘッダー表示（横並び）
- **タブレット/モバイル**: ハンバーガーメニュー化を推奨
- **背景**: サイト全体のテーマに合わせて固定
- **高さ**: 60〜80px推奨

### 1.5 実装上の注意

- [ ] ロゴクリック時にスムーズスクロール/即座遷移かを決める
- [ ] 複数テーマ(ダークモード等)対応時のアクティブ表示方法
- [ ] SEOを考慮したナビリンク構造（内部リンク最適化）

## 2. パンくずリスト仕様

### 2.1 パンくずリストの役割

ユーザーの現在位置を明確にし、階層上位のページへ迅速に戻れるようにする。

### 2.2 表示位置

- **表示箇所**: ヘッダー直下（ページコンテンツの上）またはタイトルの下
- **背景**: 薄いグレーまたは背景色との差別化
- **パディング**: 上下10px、左右15px推奨
- **フォントサイズ**: 本文より小さく（13〜14px推奨）

### 2.3 形式

- **区切り文字**: `>` または `/` を推奨
- **外観例**: `HOME / Certifications / Learning Contents`

### 2.4 ページ別パンくず例

| ページID      | ページ名      | パンくず                                        | 備考              |
|------------|-----------|---------------------------------------------|-----------------|
| P001       | トップページ    | (表示なし)                                      | トップページではパンくずは不要 |
| P002       | プロフィール    | HOME / About                                | -               |
| P003       | お問い合わせ    | HOME / Contact                              | -               |
| P005       | 資格トップ     | HOME / Certifications                       | -               |
| P005-01    | 取得資格一覧    | HOME / Certifications / List                | -               |
| P005-02    | 現在学習中     | HOME / Certifications / Current             | -               |
| P005-03-01 | 学習コンテンツ詳細 | HOME / Certifications / Contents / [コンテンツ名] | コンテンツ名は動的       |
| P006-01-01 | ゲーム詳細     | HOME / Works / Games / [ゲーム名]               | ゲーム名は動的         |

### 2.5 リンク仕様

- 最後の要素（現在ページ）は **リンクなし、テキストのみ**
- 上位要素はすべてリンク化
  - `HOME` → `/`
  - `Certifications` → `/certifications/`
  - `Contents` → `/certifications/contents/`
  - 等

### 2.6 アクセシビリティ

- [ ] `<nav>` タグで囲む
- [ ] WAI-ARIA 属性 (`aria-label="Breadcrumb"`) を使用
- [ ] 構造化データ（Schema.org BreadcrumbList）の実装検討

### 2.7 実装上の注意

- [ ] 長いパンくずの折り返し・省略対策（モバイル対応）
- [ ] 3階層以上でのパンくず表示可否判定

## 3. フッター仕様

### 3.1 フッターの役割

サイトの補足情報、関連リンク、法的表記をまとめて表示する。

### 3.2 構成要素

#### 3.2.1 セクション構成

| セクション     | 内容              | 優先度 |
|-----------|-----------------|-----|
| サイトマップリンク | 重要ページへの直接リンク    | 高   |
| 補足情報      | コピーライト、運営者情報    | 高   |
| 法的表記      | プライバシーポリシー、利用規約 | 必須  |
| SNSリンク    | 外部SNS（あれば）      | 中   |
| 更新情報      | 最終更新日またはバージョン   | 中   |

### 3.3 レイアウト

**PC（1カラム or マルチカラム例）：**

```
┌─────────────────────────────────────┐
│            フッター領域              │
├─────────────────────────────────────┤
│ [Quick Links]  [Legal]  [Social]    │
│  • About       • Privacy  • Twitter  │
│  • Works       • Contact  • GitHub   │
│  • Blog        • Terms              │
├─────────────────────────────────────┤
│ Copyright © 2026 [Your Name]        │
│ Last updated: 2026-06-13            │
└─────────────────────────────────────┘
```

**モバイル（スタック型）：**

```
┌─────────────────────────────────────┐
│            フッター領域              │
├─────────────────────────────────────┤
│ Quick Links                          │
│  • About, Works, Contact            │
│                                      │
│ Legal                                │
│  • Privacy Policy, Contact          │
│                                      │
│ Social                               │
│  • Twitter, GitHub                  │
│                                      │
│ Copyright © 2026 [Your Name]        │
│ Last updated: 2026-06-13            │
└─────────────────────────────────────┘
```

### 3.4 各セクション詳細

#### 3.4.1 Quick Links（サイトマップ）

| リンク            | URL              | 説明     |
|----------------|------------------|--------|
| HOME           | /                | トップページ |
| About          | /about/          | プロフィール |
| Certifications | /certifications/ | 資格・学習  |
| Works          | /works/          | 作品     |
| Contact        | /contact/        | お問い合わせ |
| Changelog      | /changelog/      | 変更履歴   |

#### 3.4.2 Legal（法的表記等）

| リンク | URL | 備考 |
|--------|-----|------|
| Privacy Policy | /privacy/ | 必須 |
| Contact | /contact/ | 必須 |

#### 3.4.3 Social（SNSリンク）

- GitHub（リポジトリへのリンク）
- その他SNS（アイコン + URL）

#### 3.4.4 Copyright & Update Info

```
Copyright © 2026 [Your Name]. All rights reserved.
Last updated: 2026-06-13
```

### 3.5 デザイン・レスポンシブ

- **背景色**: ヘッダーと同じまたはやや濃い色
- **テキスト色**: 可読性確保（コントラスト比 4.5:1 以上推奨）
- **パディング**: 上下30px、左右15px推奨
- **高さ**: 最小200px（コンテンツ量による）
- **PC**: 2〜3カラムレイアウト推奨
- **モバイル**: 1カラム（スタック）

### 3.6 SEO・構造化データ

- [ ] Schema.org Organization 型の実装
- [ ] `<footer>` タグで囲む
- [ ] 内部リンク最適化（rel属性の検討）

### 3.7 アクセシビリティ

- [ ] キーボード操作対応（Tab キー順序）
- [ ] ARIA ラベル（`aria-label`, `aria-describedby` など）
- [ ] リンク先テキストの明確さ（例: 「ここをクリック」NG → 「プライバシーポリシーを見る」OK）

### 3.8 実装上の注意

- [ ] フッターの "sticky" 化検討（短いページで下部に張り付けるか）
- [ ] モバイル時の折り返し・省略対策
- [ ] 最終更新日の自動更新（API/ビルド時の取得）
