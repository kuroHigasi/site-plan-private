# P006-02: 本サイト (/works/portfolio/)

## 基本情報
- 主目的: 本サイトの意図・構成・実装方針を明確に伝え、閲覧者が「どのような価値を提供するサイトか」を短時間で理解できるようにする。
- 想定ユーザー: 採用担当者、協業相手、ポートフォリオ閲覧者、技術選定の参考として閲覧する開発者。
- 想定導線: トップページ・作品一覧から遷移し、必要に応じて「使用技術」「開発プロセス」「お問い合わせ」へ移動する。

## 表示内容

| 要素名    | 詳細                                        | 備考                                                              |
|--------|-------------------------------------------|-----------------------------------------------------------------|
| 目次     | ページ内リンクで「概要/要件」「使用技術」「開発プロセス」へ移動できる目次を配置。 | セクション見出しとアンカーを一致させる。モバイルでは先頭固定ではなく通常配置として可読性を優先。                |
| 概要/要件  | 本サイトの目的、対象ユーザー、提供価値、主要ページ構成を簡潔に記載。        | 3〜5項目の箇条書きを基本にする。更新時は内容差分が分かる表現に統一する。                           |
| 使用技術   | フロントエンド、スタイリング、ビルド/運用、解析などをカテゴリ別に記載。      | 技術名のみでなく「採用理由」を1行で添える。将来の差し替え候補がある場合は注記する。                      |
| 開発プロセス | 要件整理→設計→実装→検証→改善の流れを時系列で示し、品質担保の方法を説明。    | スクリーンショット更新タイミングとドキュメント同期手順（`scripts/sync-docs-to-wiki.js`）を明記。 |

## 備考
- 参考: `01_DOCS/02_サイト構成/画面遷移図.md`、`01_DOCS/wiki/01_コンセプト定義/05_SEOとアクセシビリティ方針.md`
- 表記ルール: 技術名は正式名称で統一し、省略語を使う場合は初出で正式名称を併記する。
- 更新運用: ページ要件更新時は関連する`02_OUTPUT`の最新スクリーンショットとの差分確認を行い、不整合があれば先に要件を修正する。
